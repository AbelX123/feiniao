# feiniao 项目 Docker Compose 部署方案

## 一、架构概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           云服务器 (4核8G 起)                              │
├─────────────────────────────────────────────────────────────────────────┤
│  ┌─────────────┐                                                         │
│  │   Nginx     │  80/443 (可选，反向代理 + HTTPS)                          │
│  └──────┬──────┘                                                         │
│         │                                                                 │
│  ┌──────┴──────────────────────────────────────────────────────────────┐│
│  │                     Docker Compose 网络 (feiniao-net)                 ││
│  ├──────────────────────────────────────────────────────────────────────┤│
│  │  feiniao-users:8080  │ feiniao-dicts:8081  │ feiniao-orders:8082     ││
│  │  feiniao-recommendation:8083  │ feiniao-mcp:8084                      ││
│  ├──────────────────────────────────────────────────────────────────────┤│
│  │  mysql:3306  │  redis:6379  │  minio:9000                              ││
│  └──────────────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────────────┘
```

### 服务与端口

| 服务 | 镜像/模块 | 端口 | 说明 |
|------|----------|------|------|
| feiniao-users | feiniao-users | 8080 | 用户、创作者、认证、MinIO 文件 |
| feiniao-dicts | feiniao-dicts | 8081 | 字典（品类、标签、平台、国家等） |
| feiniao-orders | feiniao-orders | 8082 | 订单 |
| feiniao-recommendation | feiniao-recommendation | 8083 | WebSocket 聊天 + AI 推荐 |
| feiniao-mcp | feiniao-mcp | 8084 | MCP 工具（天气、模特筛选） |
| MySQL | mysql:8.0 | 3306 | 数据库 |
| Redis | redis:7-alpine | 6379 | 缓存 |
| MinIO | minio/minio | 9000/9001 | 对象存储 |

## 二、前置要求

### 2.1 服务器环境

- **系统**：Linux（推荐 Ubuntu 22.04 / CentOS 7+）
- **配置**：4 核 8G 内存、50G 磁盘（可根据流量调整）
- **网络**：开放 80、443（若用 Nginx）

### 2.2 需安装软件

```bash
# Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录使 docker 组生效

# Docker Compose
sudo apt install docker-compose-plugin   # Ubuntu
# 或
sudo yum install docker-compose-plugin   # CentOS

# Git
sudo apt install git   # Ubuntu
```

### 2.3 外部依赖

- **DeepSeek API Key**：feiniao-recommendation 对话能力
- **域名**（可选）：用于 HTTPS 和 Nginx

## 三、生产环境快速部署（仅应用模块）

当前 `docker-compose.yml` 只包含 6 个应用服务，MySQL/Redis/MinIO 需已在服务器（如 8.153.101.40）上部署并可用。

### 3.0 前置确认

1. **服务器已安装**：Docker、Docker Compose 插件（`docker compose version` 可执行）。
2. **已有 `.env`**：在项目根目录，且已填写 `MYSQL_*`、`REDIS_*`、`MINIO_*`、`DEEPSEEK_API_KEY` 等（可参考 `.env.example`）。
3. **数据库与中间件已就绪**：MySQL（3306）、Redis（6379）、MinIO（9000）在 compose 外可访问（如本机或 8.153.101.40）。

### 3.1 进入项目目录

```bash
cd /opt/feiniao   # 或你的项目根目录
```

### 3.2 构建镜像

```bash
# 构建所有 6 个服务（首次会拉依赖、编译，较慢）
docker compose build

# 仅构建某一服务（可选）
docker compose build feiniao-users
```

### 3.3 启动服务

```bash
# 后台启动
docker compose up -d

# 查看状态（应均为 running）
docker compose ps
```

### 3.4 查看日志与排错

```bash
# 所有服务日志
docker compose logs -f

# 单个服务
docker compose logs -f feiniao-users
docker compose logs -f feiniao-recommendation
```

### 3.5 验证

```bash
# 按需替换为实际 IP 或域名
curl -s http://localhost:8080/actuator/health   # users
curl -s http://localhost:8081/actuator/health   # dicts（若暴露）
curl -s http://localhost:8082/actuator/health   # orders
curl -s http://localhost:8083/                   # recommendation
curl -s http://localhost:8084/                   # mcp
curl -s http://localhost:8085/actuator/health    # payments（若暴露）
```

### 3.6 停止与更新

```bash
# 停止
docker compose down

# 更新代码后重新构建并启动
docker compose build --no-cache
docker compose up -d
```

## 四、完整部署步骤（含数据库等）

### 4.1 获取代码

```bash
cd /opt
git clone <你的仓库地址> feiniao
cd feiniao
```

### 4.2 创建环境变量文件

在项目根目录复制示例并编辑：

```bash
cp .env.example .env
vim .env   # 或使用其他编辑器
```

`.env` 内容示例：

```bash
# 数据库（必改）
MYSQL_ROOT_PASSWORD=your_root_password
MYSQL_DATABASE=feiniao
MYSQL_USER=feiniao
MYSQL_PASSWORD=your_feiniao_password

# Redis（必改）
REDIS_PASSWORD=your_redis_password

# MinIO（必改）
MINIO_ROOT_USER=minioadmin
MINIO_ROOT_PASSWORD=your_minio_password

# DeepSeek（AI 对话必需）
DEEPSEEK_API_KEY=sk-your-deepseek-api-key
```

**⚠️ 务必修改默认密码，生产环境使用强密码。.env 已加入 .gitignore，不会提交到仓库。**

### 4.3 构建镜像

本项目使用多阶段 Dockerfile，构建时会在容器内执行 Maven，**无需本地安装 Maven**：

```bash
# 构建所有服务镜像（首次较慢，需下载依赖）
docker compose build

# 若只需构建单个服务
docker compose build feiniao-users
```

> 构建脚本会依次为各模块执行 `mvn package -DskipTests`，feiniao-common、feiniao-security 等依赖会自动编译。

### 4.4 启动服务

```bash
# 前台启动（调试用）
docker compose up

# 后台启动（生产）
docker compose up -d

# 查看日志
docker compose logs -f
```

### 4.5 初始化数据库

首次部署需初始化 MySQL 库表：

```bash
# 方式一：若有 SQL 脚本
docker compose exec mysql mysql -uroot -p$MYSQL_ROOT_PASSWORD feiniao < sql/init.sql

# 方式二：手动执行建表脚本
# 将项目中的建表 SQL 导入到 feiniao 数据库
```

### 4.6 验证服务

```bash
# 检查容器状态（应均为 running）
docker compose ps

# 健康检查
curl http://localhost:8080/actuator/health          # users
curl http://localhost:8081/api/dicts/model-types   # dicts
curl http://localhost:8082/actuator/health         # orders（若启用 actuator）
curl http://localhost:8083/                        # recommendation
curl -N http://localhost:8084/sse                 # mcp (SSE 长连接)
```

## 四、核心配置文件说明

### 4.1 docker-compose.yml 结构

```yaml
services:
  mysql:      # 数据库，数据持久化
  redis:      # 缓存
  minio:      # 对象存储
  feiniao-users:
  feiniao-dicts:
  feiniao-orders:
  feiniao-recommendation:
  feiniao-mcp:
```

### 4.2 服务间依赖与网络

- 所有服务在同一 Docker 网络 `feiniao-net`
- 使用**服务名**相互访问，例如：
  - `mysql` → `jdbc:mysql://mysql:3306/feiniao`
  - `redis` → `redis://redis:6379`
  - `minio` → `http://minio:9000`
  - feiniao-mcp 调用 users：`http://feiniao-users:8080`
  - feiniao-recommendation 连接 MCP：`http://feiniao-mcp:8084`

### 4.3 环境变量映射

| 环境变量 | 影响服务 | 说明 |
|----------|----------|------|
| `SPRING_DATASOURCE_URL` | users/dicts/orders | 数据库连接 |
| `SPRING_DATA_REDIS_HOST` | users/orders | Redis 地址 |
| `MINIO_ENDPOINT` | users | MinIO 地址 |
| `DEEPSEEK_API_KEY` | recommendation | AI 对话 |
| `FEINIAO_MCP_API_USERS_BASE_URL` | mcp | users API 地址 |
| `FEINIAO_MCP_API_DICTS_BASE_URL` | mcp | dicts API 地址 |

## 五、生产环境配置建议

### 5.1 数据持久化

```yaml
volumes:
  mysql-data:    # /var/lib/mysql
  redis-data:    # /data
  minio-data:    # /data
```

务必使用 volumes，避免容器重启后数据丢失。

### 5.2 资源限制

```yaml
deploy:
  resources:
    limits:
      memory: 1G
    reservations:
      memory: 512M
```

可按实际负载调整各服务内存限制。

### 5.3 日志与重启

```yaml
logging:
  driver: "json-file"
  options:
    max-size: "50m"
    max-file: "3"

restart: unless-stopped
```

### 5.4 健康检查

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 60s
```

## 六、Nginx 反向代理（可选）

对外只暴露 80/443，由 Nginx 转发到各服务：

```nginx
# /etc/nginx/conf.d/feiniao.conf

upstream feiniao_users {
    server 127.0.0.1:8080;
}
upstream feiniao_dicts {
    server 127.0.0.1:8081;
}
upstream feiniao_orders {
    server 127.0.0.1:8082;
}
upstream feiniao_recommendation {
    server 127.0.0.1:8083;
}
upstream feiniao_mcp {
    server 127.0.0.1:8084;
}

server {
    listen 80;
    server_name your-domain.com;

    location /api/users/ {
        proxy_pass http://feiniao_users;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/dicts/ {
        proxy_pass http://feiniao_dicts;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/orders/ {
        proxy_pass http://feiniao_orders;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket 必须单独配置
    location /ws {
        proxy_pass http://feiniao_recommendation;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }

    location / {
        proxy_pass http://feiniao_recommendation;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

若使用 Docker 部署 Nginx，需将 `127.0.0.1` 改为 Compose 中的服务名（如 `feiniao-users`），并确保 Nginx 与 feiniao 在同一网络。

## 八、故障排查

### 8.1 服务无法启动

1. 查看日志：`docker compose logs feiniao-users`
2. 检查 MySQL、Redis、MinIO 是否已就绪
3. 确认环境变量与 `.env` 正确

### 8.2 服务间调用失败

1. 确认使用**服务名**（如 `mysql`、`feiniao-users`）而非 `localhost`
2. 检查是否在同一 Docker 网络
3. 检查端口是否被占用

### 8.3 WebSocket 连接失败

1. Nginx 需配置 `Upgrade`、`Connection` 等
2. 检查防火墙是否放行相应端口
3. 前端 WebSocket 地址需与 Nginx 配置一致

### 8.4 MCP 工具无响应

1. feiniao-recommendation 使用 `application-prod.yml` 中的 MCP URL：`http://feiniao-mcp:8084`
2. feiniao-mcp 使用 `application-prod.yml` 中的 users/dicts 地址
3. 检查 feiniao-mcp 日志：`docker compose logs feiniao-mcp`

### 8.5 Docker 构建失败（Linux 服务器）

若在 Linux 上构建失败，可能与 `netty-resolver-dns-native-macos`（macOS 专用）有关。可临时在 `feiniao-mcp`、`feiniao-recommendation` 的 `pom.xml` 中注释或排除该依赖后重新构建。

## 九、附录：目录结构

部署相关文件均在项目根目录：

```
feiniao/
├── 部署方案.md              # 本文档
├── docker-compose.yml       # 编排文件
├── Dockerfile               # 多阶段构建（根据 MODULE 参数构建各服务）
├── .env.example             # 环境变量示例
├── .env                     # 实际环境变量（不提交，已加入 .gitignore）
├── feiniao-*/src/main/resources/
│   ├── application-prod.yml # 生产环境配置（mcp、recommendation 有服务地址覆盖）
└── nginx/
    └── feiniao.conf         # Nginx 配置（可选）
```

### 快速部署命令汇总

```bash
cd /opt/feiniao
cp .env .env
# 编辑 .env，填入真实密码和 DEEPSEEK_API_KEY

docker compose build
docker compose up -d

# 等待 MySQL 初始化后，导入数据库脚本（若有）
docker compose ps   # 确认各服务运行正常
```
